# 阿里国际站经营定位分析技能

本技能面向**阿里巴巴国际站 ToB 商家**，根据输入信息生成一份完整的**年度经营定位分析文档**。

---

## 一、判断输入模式

收到用户消息后，首先判断属于以下哪种模式：

| 模式 | 触发条件 | 执行路径 |
|------|---------|---------|
| **A. 链接模式** | 用户提供了 alibaba.com 店铺或商品链接 | 执行第二节 → 第三节 → 第四节 → 第五节 → 第六节 |
| **B. 行业模式** | 用户说了某个行业但没有给链接 | 跳过第二节，从第三节开始，行业/类目由用户指定 |
| **C. 蓝海推荐模式** | 用户没有给链接，也没有说行业 | 执行第七节（蓝海推荐），再问用户是否继续深入分析 |

---

## 二、店铺 / 商品信息读取（链接模式）

使用 `web_fetch` 抓取链接页面内容，提取以下信息：

- **店铺名称**（公司名）
- **所在一级类目**（如：Apparel & Accessories）
- **涉及的二级/三级类目**（如：Socks、Compression Socks）
- **店铺资质**（Gold Supplier 年限、认证标识、Trade Assurance）
- **代表性产品**（列举 5-10 款，含产品名、材质、特点）
- **出口市场线索**（多语言版本、交易记录国别等）
- **响应率、响应时间、近期交易额**

> 如果用户提供的是商品链接而非店铺主页，先抓取商品所属店铺主页再补全上述信息。

信息抓取完成后，**展示给用户确认**，格式如下：

```
【店铺基础信息确认】
- 店铺名称：xxx
- 主营行业/一级类目：xxx
- 细分类目：xxx
- 代表性产品：xxx
- 认证资质：xxx
- 可见出口市场：xxx
```

然后进入第三节，**主动询问**用户补充经营信息。

---

## 三、收集用户经营信息（必须向用户提问）

展示以下问题模板，请用户逐一填写（允许用户跳过不知道的项）：

```
为了生成更精准的经营定位，请告诉我以下信息（可跳过不确定的）：

1. 公司类型：□工厂  □贸易公司  □工贸一体
2. 公司规模：员工人数约___人，年产能/年销售额约___
3. 主营产品及在店铺中的推广投放占比：
   例：压缩袜 50% | 运动袜 30% | 定制袜 20%
4. 当前主要买家国别/地区（已有客户在哪些国家）：
5. 目标开拓的新市场（希望进入哪些国家/地区）：
6. 当前店铺最大的困惑或痛点（如：询盘少、转化差、客单低等）：
7. 是否有独特的工艺/认证/专利（如：FDA、OEKO-TEX、ISO等）：
```

用户填写后，将其回答整合进后续分析。若用户跳过某项，则在对应分析中标注"待补充"并给出建议范围。

---

## 四、行业买家痛点分析

基于确认的行业类目，从以下维度系统分析**该行业 B 端买家的核心痛点**，并给出**旺铺/产品表达建议**：

### 4.1 痛点分析维度

分析时覆盖以下维度（每个维度列举 2-3 个具体痛点）：

1. **质量稳定性**：批量生产一致性、原材料品质波动、不良率控制
2. **交货周期**：打样速度、MOQ 限制、旺季备货节奏
3. **定制能力**：Logo/包装定制、颜色/规格定制、小批量起订
4. **合规认证**：目标市场准入证书（如欧盟 CE、美国 FDA、澳洲等）
5. **沟通响应**：语言障碍、时区差异、售后处理
6. **价格透明度**：阶梯报价、含税/不含税、运费报价混乱
7. **供应链风险**：独家供应商依赖、缺货断货风险

### 4.2 旺铺与产品表达建议

针对上述痛点，输出**对应的店铺文案/产品标题/主图表达策略**：

| 买家痛点 | 旺铺/产品表达建议 | 参考话术示例 |
|---------|----------------|------------|
| 质量不稳定 | 主图/详情强调品控流程 | "100% QC inspection before shipment" |
| 交货慢 | 标注现货库存和交期承诺 | "7-day sample / 30-day bulk order" |
| 认证缺失 | 首屏展示认证徽章 | "FDA / CE / OEKO-TEX Certified" |
| 定制难 | 突出定制服务流程 | "Custom logo & packaging in 500pcs MOQ" |
| ...（根据实际痛点补充） | | |

## 五、爆款款式分析（多平台）

这是本技能的**核心数据分析模块**，分两个层次进行：

### 5.1 阿里国际站爆款（询盘量最高）

使用 `web_search` 搜索以下查询，获取该类目在阿里国际站的热销/高询盘款式：
- `site:alibaba.com "[类目关键词]" most popular 2025 2026`
- `alibaba.com [类目] trending products 2026 top selling`

同时使用 `product_supplier_search` 工具搜索该类目产品，汇总：
- 搜索词：类目英文关键词（如 `compression socks, wholesale, custom`）
- 意图：`product`

**输出格式**（每个平台列出前 10-20 款，合计目标 50 款）：

| 序号 | 平台 | 商品标题（简） | 价格区间 | 商品特色亮点 | 链接 | 品牌/店铺 |
|-----|------|-------------|--------|------------|------|---------|

### 5.2 C 端平台爆款分析

针对以下 6 个 C 端平台，分别搜索 2026 年销量/增速最快的该类目产品：

**平台列表及搜索策略：**

| 平台 | 搜索方式 |
|------|---------|
| **Amazon** | `web_search`: `amazon best seller [类目] 2026 site:amazon.com` 或 `amazon [类目] #1 best seller 2026` |
| **Shopee** | `web_search`: `shopee [类目] best selling 2026` |
| **Mercado Libre** | `web_search`: `mercadolibre [类目] mas vendidos 2026` |
| **Temu** | `web_search`: `temu [类目] trending 2026 best seller` |
| **TikTok Shop** | `web_search`: `tiktok shop [类目] viral 2026 top selling` |
| **Flipkart** | `web_search`: `flipkart [类目] best seller 2026` |

每个平台搜索完成后，**汇总输出**每款商品的以下字段：

```
- 平台：
- 商品标题：
- 商品价格：
- 商品链接：（如能获取）
- 商品品牌：
- 商品特色：（材质/功能/设计亮点/差评集中点）
- 销量/排名数据：（如能获取）
```

### 5.3 爆款规律总结

分析所有平台数据后，总结出：

1. **款式规律**：哪些款式/功能/设计元素高频出现
2. **价格带分布**：主流价格区间是多少
3. **差异化机会**：哪些细分方向竞争少但增速快
4. **国别偏好**：不同市场偏好的款式差异

## 六、年度经营定位总结

综合前五节所有信息，生成一份完整的**年度经营定位报告**，结构如下：

```markdown
# [店铺名称] 2026年度经营定位报告

## 一、店铺现状概览
（基于第二节信息）
- 所在行业/类目
- 核心优势（认证/工艺/规模/响应力）
- 待提升短板

## 二、目标市场定位
（基于第三节用户信息 + 第五节平台数据）
- 主攻市场：[国家/地区] — 理由：xxx
- 次要市场：[国家/地区] — 理由：xxx
- 建议放弃/暂缓市场：xxx — 理由：xxx

## 三、产品结构定位（SKU 策略）
（基于第三节投放占比 + 第五节爆款规律）

| 产品线 | 定位角色 | 建议占比 | 核心竞争力 |
|-------|---------|---------|----------|
| [主力款] | 流量款 | 40% | 价格有竞争力，询盘转化率高 |
| [利润款] | 利润款 | 30% | 功能性/认证，溢价空间大 |
| [新品款] | 测试款 | 20% | 跟随平台趋势，快速验证 |
| [形象款] | 品牌款 | 10% | 展示工艺高度，吸引高端客户 |

## 四、店铺内容定位（旺铺/产品表达策略）
（基于第四节痛点分析）
- Banner/首屏核心卖点：xxx
- 产品标题优化方向：xxx
- 详情页必备内容模块：xxx
- 认证/品控内容强化：xxx

## 五、询盘转化提升策略
- 快速响应机制：xxx
- 样品/小单政策：xxx
- 报价透明化建议：xxx

## 六、年度重点动作（季度拆解）

| 季度 | 重点工作 | 预期目标 |
|-----|---------|---------|
| Q1（1-3月）| 旺铺改版、新品发布 | 询盘量提升20% |
| Q2（4-6月）| 旺季备货、认证申请 | 转化率提升15% |
| Q3（7-9月）| 新市场开拓 | 新增买家国别2个 |
| Q4（10-12月）| 年终促销、复购维护 | 复购率提升10% |

## 七、经营定位一句话总结
> [用一句精炼的话概括该店铺的核心定位，例如：
> "专注欧美中高端功能性袜类，以定制化服务 + 快速响应为核心壁垒，主攻 Amazon/B 端买家双轨增长"]
```

最后，将完整报告**保存为 Markdown 文件**，文件名格式：
`[店铺名]-经营定位报告-2026.md`，保存在工作目录中，并告知用户文件路径。

## 七、蓝海品类推荐模式（无链接无行业时）

当用户既未提供链接也未说明行业时，执行本节：

### 7.1 蓝海评估维度

从以下角度分析当前适合**个人创业者**的蓝海品类：

1. **市场规模**：足够大（不要太小众）但不要被大品牌垄断
2. **竞争烈度**：阿里国际站上供应商数量 < 5000，且增速快
3. **启动门槛**：MOQ 小，资金占用低，库存压力小
4. **利润空间**：毛利率 > 30%
5. **增长趋势**：Google Trends / 平台搜索量持续上升

### 7.2 推荐输出格式

```
【2026年蓝海品类推荐 - 适合个人创业者】

品类1：[品类名称]
- 所属行业：xxx
- 推荐理由：xxx（具体数据支撑）
- 目标市场：xxx
- 预估启动资金：xxx
- 主要风险：xxx
- 建议切入方式：xxx

品类2：...（共推荐 3-5 个品类）
```

推荐完成后，询问用户："您对哪个品类感兴趣？我可以为您进一步生成该品类的完整经营定位分析。"

## 八、执行规范

1. **语言**：全程使用中文与用户沟通，专业术语保留英文（如 SKU、MOQ、AOV）
2. **数据诚实性**：搜索到的数据须标注来源；无法确认的数据标注"估算"或"待核实"
3. **文档输出**：最终报告必须保存为文件（.md），不要仅在对话中展示
4. **进度提示**：每完成一个分析模块，告知用户进度（如"✓ 行业信息读取完成，正在进行买家痛点分析..."）
5. **分步确认**：在第三节收集用户信息时必须等待用户回复，不得自行假设
6. **爆款分析篇幅**：每个平台至少分析 5-10 款，总量力争覆盖 50 款，若某平台数据获取有限，明确说明并补充其他平台数量
